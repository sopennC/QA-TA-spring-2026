# Баг-репорты

---

## BUG-1

| | |
|---|---|
| **Заголовок** | `price=0` отклоняется сервером как отсутствующее поле при создании объявления |
| **Приоритет** | P1 — Major |
| **Серьёзность** | High |
| **Окружение** | https://qa-internship.avito.com |
| **Описание** | Нулевая цена — валидное значение (бесплатное объявление). Сервер не отличает `0` от отсутствующего поля и возвращает ошибку валидации. |
| **Шаги воспроизведения** | 1. `POST /api/1/item` с телом:<br>`{"name":"test","price":0,"sellerID":555555,"statistics":{"likes":1,"viewCount":1,"contacts":1}}` |
| **Фактический результат** | `400 Bad Request` — `{"result":{"message":"поле price обязательно","messages":{}},"status":"400"}` |
| **Ожидаемый результат** | `200 OK` — объявление создаётся с `price=0` |

## BUG-2

| | |
|---|---|
| **Заголовок** | Сервер принимает отрицательную цену при создании объявления |
| **Приоритет** | P1 — Major |
| **Серьёзность** | High |
| **Окружение** | https://qa-internship.avito.com |
| **Описание** | Отрицательное значение `price` не имеет смысла для объявления. Сервер не валидирует знак числа и создаёт объявление с `price=-100`. |
| **Шаги воспроизведения** | 1. `POST /api/1/item` с телом:<br>`{"name":"NegPrice","price":-100,"sellerID":555555,"statistics":{"likes":1,"viewCount":1,"contacts":1}}` |
| **Фактический результат** | `200 OK` — `{"status":"Сохранили объявление - <uuid>"}` |
| **Ожидаемый результат** | `400 Bad Request` с сообщением о невалидном значении `price` |

## BUG-3

| | |
|---|---|
| **Заголовок** | Ответ `POST /api/1/item` не соответствует контракту Postman-коллекции |
| **Приоритет** | P1 — High |
| **Серьёзность** | High |
| **Окружение** | https://qa-internship.avito.com |
| **Описание** | Postman-коллекция описывает ответ на создание как полный объект объявления. Сервер возвращает строку с UUID внутри поля `status`. |
| **Шаги воспроизведения** | 1. `POST /api/1/item` с валидными данными.<br>2. Сравнить тело ответа с описанием в Postman-коллекции. |
| **Фактический результат** | `{"status":"Сохранили объявление - bfc8b374-ad8b-46ce-8a33-cde0c23e1039"}` |
| **Ожидаемый результат** | `{"id":"...","sellerId":...,"name":"...","price":...,"statistics":{...},"createdAt":"..."}` |

## BUG-4

| | |
|---|---|
| **Заголовок** | Несоответствие регистра поля: `sellerID` в запросе и `sellerId` в ответе |
| **Приоритет** | P3 — Minor |
| **Серьёзность** | Low |
| **Окружение** | https://qa-internship.avito.com |
| **Описание** | В теле запроса поле называется `sellerID` (заглавная D), в теле ответа — `sellerId` (строчная d). |
| **Шаги воспроизведения** | 1. `POST /api/1/item` с полем `sellerID`.<br>2. `GET /api/1/item/{id}`.<br>3. Сравнить имя поля в запросе и ответе. |
| **Фактический результат** | В запросе `sellerID`, в ответе `sellerId` |
| **Ожидаемый результат** | Единое имя поля во всём API |

## BUG-5

| | |
|---|---|
| **Заголовок** | Поле `statistics` обязательно при создании, но не задокументировано как обязательное |
| **Приоритет** | P3 — Minor |
| **Серьёзность** | Low |
| **Окружение** | https://qa-internship.avito.com |
| **Описание** | При отправке запроса без поля `statistics` сервер возвращает 400. В Postman-коллекции поле присутствует в примере, но нигде не помечено как обязательное. |
| **Шаги воспроизведения** | 1. `POST /api/1/item` без поля `statistics`:<br>`{"name":"test","price":100,"sellerID":555555}` |
| **Фактический результат** | `400 Bad Request` — `{"result":{"message":"поле likes обязательно","messages":{}},"status":"400"}` |
| **Ожидаемый результат** | `statistics` помечено как обязательное в документации |

## BUG-6

| | |
|---|---|
| **Заголовок** | Поле `messages` в ответе на ошибку валидации всегда пустое |
| **Приоритет** | P3 — Minor |
| **Серьёзность** | Low |
| **Окружение** | https://qa-internship.avito.com |
| **Описание** | При ошибке валидации тело ответа содержит поле `messages`, которое всегда возвращается как `{}`. Клиент не может получить список всех ошибок за один запрос. |
| **Шаги воспроизведения** | 1. `POST /api/1/item` с несколькими невалидными полями:<br>`{"name":"","price":0,"sellerID":0,"statistics":{"likes":0,"viewCount":0,"contacts":0}}` |
| **Фактический результат** | `{"result":{"message":"поле name обязательно","messages":{}},"status":"400"}` |
| **Ожидаемый результат** | `messages` содержит детализацию по каждому невалидному полю |
